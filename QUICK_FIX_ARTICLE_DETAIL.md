# 快速修复 article-detail 页面依赖分析错误

## ⚡ 快速解决方案（3步）

### 步骤 1: 关闭"过滤无依赖文件"功能

1. 打开微信开发者工具
2. 点击右上角 **设置** 图标（⚙️）
3. 选择 **项目设置**
4. 在 **本地设置** 中，**取消勾选** "过滤无依赖文件"
5. 关闭并重新打开微信开发者工具

### 步骤 2: 清理缓存

1. 在微信开发者工具中：**工具** → **清除缓存** → **清除全部**
2. 等待清理完成

### 步骤 3: 重新编译

1. 点击 **编译** 按钮
2. 等待编译完成
3. 检查控制台，错误应该消失

## ✅ 验证修复

1. 打开心理库页面
2. 点击任意文章
3. 应该能正常跳转到文章详情页
4. 控制台不应该再有依赖分析错误

## 🔧 如果仍然有问题

### 方法 A: 检查 project.config.json

确保 `project.config.json` 中包含以下配置：

```json
{
  "setting": {
    "ignoreDevUnusedFiles": false,
    "ignoreUploadUnusedFiles": false
  }
}
```

### 方法 B: 手动添加页面到编译列表

在微信开发者工具中：
1. 点击 **详情** 按钮
2. 选择 **本地设置** 标签
3. 在 **编译设置** 中，确保 **"过滤无依赖文件"** 未勾选

### 方法 C: 重启微信开发者工具

1. 完全关闭微信开发者工具
2. 重新打开
3. 重新打开项目
4. 重新编译

## 📝 说明

这个错误是微信开发者工具的代码依赖分析功能导致的。即使页面在 `pages.json` 中正确注册，如果开启了"过滤无依赖文件"功能，工具可能会忽略动态引用的页面。

**最佳实践**：在开发阶段，建议关闭"过滤无依赖文件"功能，这样可以避免类似问题。
